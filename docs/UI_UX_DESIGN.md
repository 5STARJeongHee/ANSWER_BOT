# 사내 Slack 챗봇 UI/UX 설계 문서

- 작성일: 2026-06-16
- 버전: 1.0
- 대상: 개발팀 / 운영팀

---

## 1. 설계 원칙

| 원칙 | 설명 |
|------|------|
| 스레드 우선 | 모든 봇 응답은 원래 메시지 스레드에 달린다. 채널 타임라인을 오염시키지 않는다. |
| 불확실 시 명시 | 확신 없는 답변은 절대 단정적으로 전달하지 않는다. fallback으로 담당자를 연결한다. |
| 즉각 피드백 | 3초 이내에 "분석 중" 메시지로 처리 중임을 알린다. |
| AI 투명성 | 모든 답변에 "AI 생성 답변" 컨텍스트 배지를 포함한다. |
| 근거 기반 신뢰도 | 신뢰도를 임의 수치로 표시하지 않고, RAG에서 참고한 과거 대화 건수로 표시한다. |

## 2. 사용자 인터랙션 플로우

```
┌─────────────────────────────────────────────────────────────────┐
│                     사용자 인터랙션 전체 흐름                       │
└─────────────────────────────────────────────────────────────────┘

[사용자] Slack 채널에서 @챗봇 멘션 또는 메시지 작성
         │
         ▼
[이벤트 핸들러] Slack ack() 즉시 반환 (3초 이내)
         │
         ├─── 빈 질문 ──────────────────────────────────────────▶ [환영 메시지] 전송
         │
         ▼
[UI] "분석 중..." Block Kit 메시지 → 스레드에 즉시 전송
         │
         ▼  (백그라운드 스레드)
[저장] 사용자 메시지 DB 저장 + 임베딩 생성
         │
         ▼
[RAG] pgvector에서 유사 과거 대화 검색 (상위 5건)
         │
         ▼
[LLM] 답변 초안 생성 (OpenRouter)
         │
         ▼
[평가] Fallback 평가 프롬프트 실행
         │
         ├─── can_answer_directly: false ───────────────────────▶ [Fallback 메시지]
         │                                                          "분석 중" 메시지를 업데이트
         │                                                          담당자 멘션 + 질문 인용
         │
         ├─── LLM 오류 / 타임아웃 ──────────────────────────────▶ [에러 메시지]
         │                                                          "분석 중" 메시지를 업데이트
         │
         └─── can_answer_directly: true ─────────────────────▶  [일반 답변 메시지]
                                                                   "분석 중" 메시지를 업데이트
                                                                   참고 과거 대화 건수 표시
                                                                        │
                                                                        ▼
                                                              [피드백 이모지 자동 추가]
                                                              👍 / 👎 시드 이모지
                                                                        │
                                                                        ▼
                                                              [reaction_added 이벤트]
                                                              사용자 리액션 → 피드백 로그
```

## 3. Block Kit 컴포넌트 설명

### 3.1 로딩 메시지 (분석 중)

**파일**: `slack_bot/ui/message_blocks.py` → `build_thinking_blocks()`

**목적**: 사용자에게 처리 중임을 3초 이내에 알린다.

**구성**:
```
┌─────────────────────────────────────────────┐
│ ⏳ 분석 중입니다...                           │
│ 잠시만 기다려 주세요.                          │
└─────────────────────────────────────────────┘
```

- section 블록 1개로 단순하게 구성 (복잡한 블록 불필요)
- 이후 답변 완료 시 `chat_update`로 교체됨

---

### 3.2 일반 답변 메시지

**파일**: `slack_bot/ui/message_blocks.py` → `build_answer_blocks()`

**목적**: RAG 기반 LLM 답변을 신뢰도 정보와 함께 전달한다.

**구성**:
```
┌─────────────────────────────────────────────────────────────────┐
│ 답변 내용 (section 블록)                                          │
│                                                                   │
│ 2~5문장 내 핵심 내용                                               │
├─────────────────────────────────────────────────────────────────┤
│ 🔍 참고한 과거 대화: N건  |  🤖 AI 생성 답변 — 중요한 내용은      │
│    담당자에게 확인하세요.                    (context 블록)         │
├─────────────────────────────────────────────────────────────────┤
│ 👍 / 👎 이모지로 답변 품질을 알려주세요.      (context 블록)        │
└─────────────────────────────────────────────────────────────────┘
```

**신뢰도 표시 방식**:
- `context_count > 0`: "참고한 과거 대화: N건" → 근거가 있는 답변
- `context_count == 0`: "관련 기록 없이 생성된 답변" → 사용자에게 주의 환기

임의 확률(%)을 표시하지 않는다. 실제 RAG 검색 결과 수만 사용한다.

---

### 3.3 Fallback 메시지 (담당자 호출)

**파일**: `slack_bot/ui/message_blocks.py` → `build_fallback_blocks()`

**목적**: 불확실한 답변 대신 담당자를 연결한다.

**구성**:
```
┌─────────────────────────────────────────────────────────────────┐
│ ⚠️ 정확한 답변을 드리기 어렵습니다.                               │
│ 해당 질문은 사실 확인이 필요하거나 최신 정보가 부족합니다.          │
├─────────────────────────────────────────────────────────────────┤
│ 질문 내용                                                         │
│ >>> [원본 질문 인용 - 최대 200자]                                  │
├─────────────────────────────────────────────────────────────────┤
│ (divider)                                                         │
├─────────────────────────────────────────────────────────────────┤
│ 담당자 @홍길동에게 문의해 주세요.                                   │
├─────────────────────────────────────────────────────────────────┤
│ 🤖 AI 챗봇이 자동 생성한 메시지입니다.       (context 블록)        │
└─────────────────────────────────────────────────────────────────┘
```

**중요**: Fallback 발동 시 "분석 중" 메시지를 `chat_update`로 교체한다. 별도 메시지를 추가하지 않아 채널이 지저분해지지 않는다.

---

### 3.4 에러 메시지

**파일**: `slack_bot/ui/message_blocks.py` → `build_error_blocks()`

**목적**: 시스템 오류 발생 시 빈 응답 없이 사용자에게 안내한다.

**구성**:
```
┌─────────────────────────────────────────────────────────────────┐
│ ❌ 일시적인 오류가 발생했습니다.                                   │
│ 잠시 후 다시 시도해 주세요. 문제가 지속되면 관리자에게 문의하세요.  │
├─────────────────────────────────────────────────────────────────┤
│ 🤖 AI 챗봇 서비스 — 오류 발생 시 담당자에게 알려주세요. (context) │
└─────────────────────────────────────────────────────────────────┘
```

---

### 3.5 환영 메시지 (빈 멘션)

**파일**: `slack_bot/ui/message_blocks.py` → `build_greeting_blocks()`

**목적**: 질문 없이 @봇만 멘션했을 때 사용법을 안내한다.

**구성**:
```
┌─────────────────────────────────────────────────────────────────┐
│ 👋 안녕하세요! 사내 Q&A 챗봇입니다.                               │
│ 업무 관련 궁금한 점을 질문해 주세요.                               │
│ 과거 대화를 바탕으로 답변해 드립니다.                              │
├─────────────────────────────────────────────────────────────────┤
│ 예시: `@챗봇 연차 신청 방법이 어떻게 되나요?`    (context 블록)    │
└─────────────────────────────────────────────────────────────────┘
```

## 4. 응답 유형별 메시지 업데이트 패턴

모든 경로에서 "분석 중" 메시지를 `chat_update`로 덮어쓴다. 추가 메시지를 올리지 않는다.

```
thinking_ts (분석 중 메시지 ts)
         │
         ├─── 답변 완료 ──────▶ update_message_blocks(thinking_ts, answer_payload)
         ├─── Fallback ────────▶ update_message_blocks(thinking_ts, fallback_payload)
         └─── 오류 ───────────▶ update_message_blocks(thinking_ts, error_payload)
```

**이점**: 채널 스레드 내 메시지 수가 최소화된다. 사용자가 처리 흐름을 자연스럽게 인지한다.

## 5. 피드백 수집 흐름 (F-08, Phase 2)

```
[봇 답변 전송 완료]
         │
         ▼
[reactions_add API] 👍 / 👎 시드 이모지 자동 추가
(reactions:write 스코프 필요)
         │
         ▼
[사용자] 👍 또는 👎 클릭 (혹은 추가 이모지)
         │
         ▼
[reaction_added 이벤트 수신]
         │
         ├─── 봇 메시지인지 확인 (conversations_history 조회)
         ├─── 시드 이모지가 아닌 사용자 리액션인지 확인
         └─── 피드백 계열 이모지인지 확인 (👍/👎 계열)
                  │
                  ▼
         [현재] 피드백 로그 기록
         [Phase 2] DB에 feedback 레코드 저장
```

**필터링 규칙**:
- `_POSITIVE_REACTIONS`: `thumbsup`, `+1`, `white_check_mark`
- `_NEGATIVE_REACTIONS`: `thumbsdown`, `-1`, `x`
- 봇이 아닌 메시지(사용자 메시지)에 달린 리액션은 무시
- 피드백 계열이 아닌 이모지는 무시

## 6. Slack 스코프 요구사항

| 스코프 | 용도 | 필수 여부 |
|--------|------|---------|
| `chat:write` | 메시지 전송 및 업데이트 | 필수 |
| `reactions:write` | 👍/👎 시드 이모지 자동 추가 | 피드백 기능 사용 시 필수 |
| `reactions:read` | reaction_added 이벤트 수신 | 피드백 기능 사용 시 필수 |
| `channels:history` | 봇 메시지 여부 확인 (conversations_history) | 피드백 기능 사용 시 필수 |
| `app_mentions:read` | @봇 멘션 이벤트 수신 | 필수 |

## 7. 접근성 고려사항

### 7.1 스크린리더 / 알림 텍스트

Slack의 Block Kit은 시각적 블록과 별도로 `text` 필드를 알림 텍스트로 사용한다. 모든 Block Kit 페이로드에 `text` 필드를 필수로 포함하며, 블록 내용의 핵심을 일반 텍스트로 요약한다.

```python
# 항상 이 형태로 반환
return {
    "text": "일반 텍스트 요약 (알림, 스크린리더용)",
    "blocks": [...]
}
```

### 7.2 이모지 사용 원칙

- 이모지는 의미를 보완하는 용도로만 사용한다. 이모지 단독으로 의미를 전달하지 않는다.
- 예: `:warning:` 뒤에 반드시 텍스트 설명이 따라온다.
- 모바일 Slack에서도 이모지가 표시되지 않을 때 텍스트만으로 의미가 전달되어야 한다.

### 7.3 텍스트 길이 제한

| 블록 유형 | 최대 길이 | 처리 |
|-----------|---------|------|
| section text | 3,000자 | 말줄임표 처리 (`_truncate()`) |
| 질문 인용 | 200자 | 슬라이싱 |
| context 블록 | 2,000자 | Slack 내부 제한 |

### 7.4 모바일 Slack 고려사항

- 모든 블록을 수직으로 쌓는 단일 컬럼 레이아웃 사용 (fields 속성 사용 금지)
- 버튼 액션은 CTA가 필요할 때만 사용하고, 현재는 URL 없는 버튼 대신 텍스트 멘션 사용
- context 블록 텍스트는 보조 정보이므로 짧게 유지

## 8. 파일 구조

```
slack_bot/
├── ui/
│   ├── __init__.py
│   ├── message_blocks.py     # Block Kit 페이로드 팩토리 함수

│   └── slack_service.py      # Block Kit 통합 메시지 전송 함수 추가

```
